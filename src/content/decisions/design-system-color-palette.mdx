## Color Selection Process

We started by auditing existing colors across all products—we found 47 different blues alone! This highlighted the need for a systematic approach.

The selection process involved:

1. **Brand alignment**: Started with our primary brand color
2. **Accessibility testing**: Ensured all combinations met WCAG AA
3. **Semantic mapping**: Assigned meaning to each color family
4. **Shade generation**: Used HSL to create consistent 10-shade scales

## The Final Palette

- **Brand (Blue)**: Primary actions, links, focus states
- **Success (Green)**: Positive feedback, completion, growth
- **Warning (Amber)**: Caution, pending states, attention
- **Error (Red)**: Errors, destructive actions, alerts
- **Neutral (Gray)**: Text, backgrounds, borders

Each color has shades from 50 (lightest) to 900 (darkest).

## Accessibility Considerations

We made accessibility non-negotiable:

- All text colors meet 4.5:1 contrast ratio minimum
- Interactive elements meet 3:1 against backgrounds
- Color is never the only indicator of meaning
- Tested with color blindness simulators

## Implementation

The colors are available as:

- **Design tokens** in Figma with semantic naming
- **CSS custom properties** for web
- **Swift/Kotlin constants** for mobile
- **Tailwind config** for utility classes

This ensures consistency from design to code.
